Breakpoints

Responsive styling can be found in the Sass file _responsive.scss. WorkAround is built mobile-first. The following components have defined breakpoints in order to maximize responsiveness: navigation, text, and the grid system. Change you screen size to try out the various breakpoints on the examples that ensue.

Navigational items float right by default at screen widths greater than 480px. At smaller screen widths, the navigation bar items stack on top of each other bottom up. See _responsive.scss lines 2-9 for the definition of this breakpoint or the Navigation page for more specifics.


The text of a page should automatically adjust to fit the screen size. For screen widths of less than 780px, h1 text adjusts in font size and margin. Likewise, for screen widths of less than 480px, so does h2 text. See _responsive.scss lines 26-30 and 11-13 respectively or the Text page for more specifics.

For the grid system, at screen widths below 970px a 6-column grid wraps onto 2 lines of 3 columns each. Until a screen width below 640px is reached at which point all size grids stack their columns in consecutive order into a single column. See _responsive.scss lines 16-24 and 32-36 respectively or the Grid page for more specifics.
